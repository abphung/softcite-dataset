Coding Scheme

Code Phase OrderExample
software_not_used 1 1For mentions of software that the authors didn't use (e.g., they discuss why they didn't use it). This code should be used for software that the authors considered (e.g., the competitors of the software they used).
software_name 1 1whatever the name is
publication 1 1Reference to a paper primarily detailing the software itself (cited because it uses or introduces the software)
users_guide 1 1Reference to a user’s manual for the software
software_used 1 1For mentions of software that was used in the research
in-text_mention 1 1Mention of software in the text part of the paper
reference 1 1A software relevant reference. If multiple references are present in the coding unit and refer to the same piece of software, multiple has_reference codes should be attached to the same in-text_mention, not broken up.
has_selection 1 1FIXME
project_page 1 1URL for the software’s online presence - downloads, orders, FAQs
creator 1 1A mention of the creator of the software, don't include names as citation anchors. Haven't decided what to do about references here yet. Example of where to use: Software, written by xyz, was used for ... 'coupled to MetaMorph imaging software (Universal Imaging Corporation, Downingtown, PA).' Code just the company name
creator 1 1unlike in the rest of the text, code the author names of the paper, and include the full string of multiple authors as a single value for a :creator code
date 1 1A date used to indicate a version of the software. Not the date of the paper or reference. If use date as version number include both codes
version_number 1 1whatever the version number is. If use date as version number include both codes
